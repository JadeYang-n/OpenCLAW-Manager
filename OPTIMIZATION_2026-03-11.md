### 2. 部署方式说明优化（面向 Windows 用户）

**问题**: 原有说明文字不够清晰，没有针对 Windows 用户的场景说明

**优化内容**:

#### 新增选择指南
```
💡 如何选择？
• 第一次使用 或 只想简单快捷 → 选择「本地直接安装」
• 有 Linux 经验 或计划后续部署到服务器 → 选择「WSL2 部署」
• 有 Docker 经验 或需要环境隔离 → 选择「Docker 部署」
```

#### 每种方式详细说明

**本地直接安装** 🖥️（推荐）
- **适合**: 大多数 Windows 用户，尤其是第一次使用 OpenCLAW 的用户
- **不适合**: 需要 Linux 特定功能或计划部署到 Linux 服务器的用户
- **优点**:
  - ✅ 安装最简单，一键完成
  - ✅ 性能最好，无虚拟化开销
  - ✅ 资源占用少，运行流畅
  - ✅ 无需学习额外工具
  - ✅ 文件路径兼容性好（Windows 路径）
- **缺点**:
  - ❌ 部分 Linux 专属工具可能不兼容
  - ❌ 迁移到其他系统需要重新配置

**WSL2 部署** 🐧
- **适合**: 有 Linux 使用经验、需要在 Windows 上模拟 Linux 环境的开发者
- **不适合**: 不熟悉命令行、只想简单使用的用户
- **优点**:
  - ✅ 完整的 Linux 环境，兼容性好
  - ✅ 接近生产环境部署
  - ✅ 可使用所有 Linux 工具和脚本
  - ✅ 便于后续迁移到 Linux 服务器
- **缺点**:
  - ❌ 需要启用 WSL2 并安装 Linux 发行版
  - ❌ 占用更多磁盘空间和内存
  - ❌ 文件路径需要转换（/mnt/c/...）
  - ❌ 学习成本较高

**Docker 部署** 🐳
- **适合**: 有 Docker 经验、需要环境隔离或频繁迁移的用户
- **不适合**: 没有 Docker 基础、追求简单安装的用户
- **优点**:
  - ✅ 环境完全隔离，不影响系统
  - ✅ 便于迁移和备份（镜像导出）
  - ✅ 版本管理清晰，可随时回滚
  - ✅ 开发和生产环境一致
  - ✅ 多实例部署方便
- **缺点**:
  - ❌ 需要安装 Docker Desktop
  - ❌ 学习成本最高
  - ❌ 资源占用较大
  - ❌ 文件路径需要映射配置
