### 2. 实例创建时 Skills 显示"暂无可用"
**问题**: 即使有已安装的 Skill，创建实例时仍显示"暂无可用 Skills"

**原因**:
- `loadSkills()` 函数是空的（TODO 注释）
- 没有调用后端 API
- 错误处理缺失

**解决方案**:
```typescript
async function loadSkills() {
  try {
    const token = getToken() || ''
    const list = await invoke<any[]>('list_skills', { token })
    setSkills(list)
  } catch (error) {
    // 使用模拟数据作为 fallback
    setSkills([...])
  }
}
```

**修改文件**:
- `apps/web/src/modules/instances/InstancesPage.tsx` - 实现 Skills 加载

### 3. 环境检测失败：undefined
**错误信息**: `环境检测失败：undefined`

**原因**:
- 前端调用 `check_environment()` 没有传递 `mode` 参数
- 后端 API 签名要求 `mode: DeployMode` 参数
- 参数不匹配导致返回 undefined

**解决方案**:
```typescript
// 添加 mode 参数（默认检测 Windows 原生环境）
const result = await invoke<EnvCheckReport>('check_environment', { mode: 'windows' })
```

**修改文件**:
- `apps/web/src/modules/setup/SetupPage.tsx` - 添加 mode 参数

## 🔧 技术细节

### SQLite 表迁移

SQLite 的 `ALTER TABLE` 支持有限：
- ✅ 可以添加列（但旧行该列为 NULL）
- ❌ 不能删除列
- ❌ 不能修改列类型
- ❌ 不能添加约束

**标准迁移流程**:
```sql
-- 1. 创建新表
CREATE TABLE new_table (...);

-- 2. 复制数据
INSERT INTO new_table SELECT ... FROM old_table;

-- 3. 删除旧表
DROP TABLE old_table;

-- 4. 重命名
ALTER TABLE new_table RENAME TO old_table;
```

### 错误处理最佳实践

```typescript
  setData(result)

  setFallbackData()
}
```
