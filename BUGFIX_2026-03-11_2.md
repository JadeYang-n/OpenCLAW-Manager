## 🐛 问题清单

### 问题 1：环境检测逻辑错误 ✅ 已修复

**问题描述**:
- 环境检测功能是通用的，检测所有环境项
- 用户已经成功部署，但环境检测显示"pnpm 未安装"
- 环境检测和一键部署并列，不符合用户操作流程

**根本原因**:
环境检测没有根据部署方式过滤，检测了不需要的项目。

**解决方案**:
重构部署向导流程：
```
1. 选择部署方式
   ↓
2. 环境检测（只检测该方式需要的环境）
   ↓
3. 一键部署
```

**实现细节**:
- 定义三种部署方式的检测项配置：
  - **Windows 原生**: Node.js、pnpm、Git
  - **WSL2**: WSL2 功能、Linux 发行版、Node.js (WSL)、pnpm (WSL)
  - **Docker**: Docker Desktop、Docker 服务

- 使用步骤指示器引导用户
- 环境检测隶属于部署流程，不再是独立功能

**修改文件**:
- `apps/web/src/modules/setup/SetupPage.tsx` - 完全重构

### 问题 2：Skill 安装不能真正实现 ✅ 已修复

**问题描述**:
点击 Skill 安装按钮后，无法真正实现安装。

**原因**:
后端 `install_skill` API 返回 `Result<(), String>`，没有返回成功消息。

**修复**:
```rust
// 修复前
pub fn install_skill(...) -> Result<(), String> {
    // ...
    Ok(())
}

// 修复后
pub fn install_skill(...) -> Result<String, String> {
    // ...
    Ok(format!("✅ Skill {} 安装成功", req.skill_id))
}
```

**修改文件**:
- `src-tauri/src/commands/skills.rs`

### 问题 3：创建实例报错 ✅ 已修复

**错误信息**:
```
❌ 创建失败
解析响应失败：error decoding response body: expected value at line 1 column 1
```

**原因分析**:
1. 前端传递参数方式错误：展开对象后删除字段，导致格式不匹配
2. 前端在创建后又单独调用绑定部门的 API，但后端已经处理了

**错误代码**:
```typescript
const req = { ...createForm }
const deptId = req.department_id
delete req.department_id  // ❌ 删除字段

const instanceId = await invoke('create_instance', { token, req })

// 又单独绑定部门
if (deptId) {
  await deptApi.bindInstanceToDepartment(token, { instance_id: instanceId, department_id: deptId })
}
```

**修复代码**:
```typescript
// 直接传递完整对象
const req = {
  name: createForm.name,
  endpoint: createForm.endpoint,
  api_key: createForm.api_key,
  config_id: createForm.config_id || null,
  skill_ids: createForm.skill_ids || [],
  department_id: createForm.department_id || null,
}

const instanceId = await invoke<string>('create_instance', { token, req })
// 后端已经处理所有绑定逻辑，无需前端再调用
```

**修改文件**:
- `apps/web/src/modules/instances/InstancesPage.tsx`
