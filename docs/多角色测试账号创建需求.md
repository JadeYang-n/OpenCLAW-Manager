# 多角色测试账号创建需求

**版本：** v2.0  
**日期：** 2026-04-06  
**优先级：** P0（核心功能）  
**目的：** 创建多角色测试账号，支持登录测试，数据持久化保存

---

## 📋 需求澄清（2026-04-06 更新）

### 核心需求

1. **不是做"切换账号"功能** - 而是**创建测试账号**，让用户逐个登录测试
2. **测试数据要持久化** - 不能切换账号后数据丢失，要能验证权限隔离
3. **财务角色权限修正** - 财务 = 个人权限 + 全局用量只读

### 测试场景

```
测试流程：
1. 登录 test_employee → 创建实例、上传 Skill → 退出
2. 登录 test_dept_head → 应该能看到本部门员工的实例 → 审核 Skill → 退出
3. 登录 test_admin → 查看所有数据 → 退出
4. 登录 test_finance → 查看全局用量（只读） → 退出

关键：每个账号操作的数据要保存，不能丢失！
```

## 🔧 技术实现

### 1. CLI 命令实现

```rust
// src/server/src/cli/commands/test_accounts.rs

use crate::db::{create_user, create_department};
use crate::models::{Role, User, Department};
use bcrypt::hash;

pub fn create_test_accounts() -> Result<()> {
    println!("🔧 开始创建测试账号...\n");
    
    // 1. 创建测试部门
    let dept_dev = create_department(Department {
        id: "dept_dev".to_string(),
        name: "研发部".to_string(),
    })?;
    
    let dept_finance = create_department(Department {
        id: "dept_finance".to_string(),
        name: "财务部".to_string(),
    })?;
    
    // 2. 创建测试账号
    let password_hash = hash("Test123!", 12)?;
    
    let accounts = vec![
        ("test_admin", Role::Admin, None),
        ("test_ops", Role::Ops, None),
        ("test_dept_head", Role::DeptHead, Some(dept_dev.id.clone())),
        ("test_employee", Role::Employee, Some(dept_dev.id.clone())),
        ("test_finance", Role::Finance, Some(dept_finance.id.clone())),
    ];
    
    for (username, role, dept_id) in accounts {
        match create_user(User {
            id: format!("user_{}", username),
            username: username.to_string(),
            password_hash: password_hash.clone(),
            role,
            department_id: dept_id,
            is_test_account: true,
        }) {
            Ok(_) => println!("✅ 创建账号：{} ({})", username, role),
            Err(e) => println!("❌ 创建账号 {} 失败：{}", username, e),
        }
    }
    
    // 3. 输出账号列表
    println!("\n✅ 测试账号创建成功！\n");
    println!("账号列表：");
    println!("┌───────────────────┬───────────┬────────────┬─────────┐");
    println!("│ 账号名            │ 密码      │ 角色       │ 部门    │");
    println!("├───────────────────┼───────────┼────────────┼─────────┤");
    println!("│ test_admin        │ Test123!  │ 管理员     │ -       │");
    println!("│ test_ops          │ Test123!  │ 运维       │ -       │");
    println!("│ test_dept_head    │ Test123!  │ 部门负责人 │ 研发部  │");
    println!("│ test_employee     │ Test123!  │ 普通员工   │ 研发部  │");
    println!("│ test_finance      │ Test123!  │ 财务       │ 财务部  │");
    println!("└───────────────────┴───────────┴────────────┴─────────┘");
    println!("\n提示：");
    println!("- 测试账号仅限开发/测试环境使用");
    println!("- 生产环境请手动删除测试账号");
    println!("- 使用方式：用以上账号逐个登录 OCM Manager 测试权限\n");
    
    Ok(())
}
```

### 2. 数据持久化保证

```rust
// 所有测试账号的数据都保存在正式数据库中
// 通过 owner_id 和 department_id 区分归属

// 实例表结构
CREATE TABLE instances (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    owner_id TEXT NOT NULL,          -- 创建者 ID
    department_id TEXT,               -- 所属部门
    -- ...其他字段
);

// 查询时根据角色过滤
SELECT * FROM instances
WHERE 
    -- 管理员：无限制
    (user.role = 'admin')
    OR
    -- 运维：无限制
    (user.role = 'ops')
    OR
    -- 部门负责人：本部门
    (user.role = 'dept_head' AND instances.department_id = user.department_id)
    OR
    -- 员工：自己的
    (user.role = 'employee' AND instances.owner_id = user.id)
```
