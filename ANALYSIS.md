# 功能边界与用户角色分析

## 一、功能边界检查

### 现有 10 个模块详细分析

| 模块 | 核心功能 | 目标用户 | 使用频率 |
|------|----------|----------|----------|
| **instances** | 实例列表、创建、删除、批量操作 | 运维 | 每天 |
| **dashboard** | 状态卡片、Token 今日/本月、7 天趋势 | 管理者 | 每周 |
| **monitor** | Token 详细分析、模型分布、预算设置 | 财务/运维 | 每周 |
| **config** | 配置编辑、保存、同步 | 运维 | 偶尔 |
| **setup** | 环境检测、一键部署 | 运维 | 偶尔 |
| **deployment** | 部署向导（与 setup 重叠） | 运维 | 偶尔 |
| **skills** | Skills 列表、安装、卸载 | 运维/开发 | 偶尔 |
| **security** | 密钥管理、安全体检、一键加固 | 安全/运维 | 每月 |
| **audit** | 审计日志查看、筛选 | 审计/管理者 | 出事时 |
| **settings** | 主题、语言、路径配置 | 所有人 | 偶尔 |

## 三、用户角色分析

### 目标用户画像

```
🏢 企业使用 OpenCLAW Manager 的典型场景：

某电商公司部署了 5 个 OpenCLAW 实例
├── 客服部：2 个实例（售前咨询、售后服务）
├── 销售部：1 个实例（销售机器人）
├── 产品部：1 个实例（内部助手）
└── 测试环境：1 个实例

涉及人员：
├── 老板/CEO：关心花了多少钱、ROI 如何
├── CTO：关心系统稳不稳定、有没有安全风险
├── 运维负责人：负责部署、监控、排障
├── 各部门 AI 使用人员：日常使用 OpenCLAW
└── 财务：审核 AI 成本预算
```

## 四、权限体系设计

### 角色定义

| 角色 | 典型职位 | 核心诉求 | 权限范围 |
|------|----------|----------|----------|
| **超级管理员** | CTO/技术总监 | 全局掌控 | 所有功能 + 用户管理 |
| **运维管理员** | 运维负责人 | 稳定运行 | 部署、监控、配置、排障 |
| **查看员** | 部门主管 | 看数据 | 只看仪表盘、Token 统计 |
| **审计员** | 财务/合规 | 审计追溯 | 只看审计日志、成本报表 |
| **普通用户** | AI 使用人员 | 使用 OpenCLAW | 无权访问 Manager（只用 OpenCLAW） |

## 五、各角色看到的面板

### 1️⃣ 超级管理员（CTO）

```
📊 工作台
├── 实例管理        ✅ 全部权限
├── 全局仪表盘      ✅ 全部权限

⚙️ 运维中心
├── 部署向导        ✅
├── 配置管理        ✅
├── Skills          ✅
├── Token 分析      ✅

🔐 安全与设置
├── 审计日志        ✅
├── 安全中心        ✅
├── 系统设置        ✅
├── 用户管理        ✅ 独有功能
```

**仪表盘内容**：
- 所有实例状态总览
- 全公司 Token 消耗（今日/本月/趋势）
- 成本分摊（按部门）
- 告警摘要
- 系统健康度评分

---

### 2️⃣ 运维管理员

```
📊 工作台
├── 实例管理        ✅ 操作权限
└── 全局仪表盘      ✅ 查看权限

⚙️ 运维中心
├── 部署向导        ✅
├── 配置管理        ✅
├── Skills          ✅
├── Token 分析      ✅ 查看权限

🔐 安全与设置
├── 审计日志        ✅ 查看权限（自己操作的）
├── 安全中心        ✅
├── 系统设置        ⚠️ 部分权限（不能改用户管理）
```

**仪表盘内容**：
- 实例状态（详细）
- 资源使用率（CPU/内存）
- 告警列表
- 最近操作记录

**不能访问**：
- ❌ 用户管理
- ❌ 成本分摊（只能看总量）
- ❌ 审计日志（只能看自己的）

---

### 3️⃣ 查看员（部门主管）

```
📊 工作台
├── 实例管理        ❌ 只读（看自己部门的）
└── 全局仪表盘      ✅ 只读

⚙️ 运维中心
├── 部署向导        ❌
├── 配置管理        ❌
├── Skills          ❌
└── Token 分析      ✅ 只看自己部门的

🔐 安全与设置
├── 审计日志        ❌
├── 安全中心        ❌
└── 系统设置        ❌
```

**仪表盘内容**：
- 本部门实例状态
- 本部门 Token 消耗
- 本部门成本

**不能访问**：
- ❌ 任何修改操作
- ❌ 其他部门数据
- ❌ 系统配置

---

### 4️⃣ 审计员（财务/合规）

```
📊 工作台
├── 实例管理        ❌
└── 全局仪表盘      ⚠️ 只看成本相关

⚙️ 运维中心
├── 部署向导        ❌
├── 配置管理        ❌
├── Skills          ❌
└── Token 分析      ✅ 全部（成本报表）

🔐 安全与设置
├── 审计日志        ✅ 全部权限
├── 安全中心        ❌
└── 系统设置        ❌
```

**仪表盘内容**：
- 成本报表（按部门/项目）
- 预算执行情况
- 异常消耗告警

**不能访问**：
- ❌ 技术配置
- ❌ 部署操作

---

### 5️⃣ 普通用户（AI 使用人员）

```
❌ 无权访问 OpenCLAW Manager

他们直接使用 OpenCLAW 的聊天界面，不需要管理功能。
```
